# T-9005 System Impact Map

This is a round-one compatibility map for the proposed closed, mission-locked
Atrahasis redesign.

## Preserve With Minor Reframing

- `C38` FSPA
- `C39` LCML
- `C40` DAAF
- `C44` constrained generation
- `C45` sovereign server framework
- `C47` cross-compilation forge

Reason:
These specs already support sovereignty, native control, and bounded authority.
They mainly need doctrine-language cleanup and explicit release-class hooks.

## Preserve With New Release-Class Addenda

- `C23` runtime
- `C24` habitat / federation
- `C36` interface membrane
- `C41` manifest authority
- `C42` tool authority
- `C33` operations
- `C34` recovery
- `C35` sentinel
- `C37` feedback fabric

Reason:
These layers are useful in a closed system, but they need explicit awareness
of:

- `SANCTUM_INTERNAL` vs external surfaces,
- capability embargo,
- release adjudication,
- and internal-only high-consequence flows.

## Major Rewrite Required

- `C14` AiBC governance
- `C15` AIC economics
- `C18` funding and business operations
- `C22` implementation planning

Reason:
These are where the current repo most strongly assumes:

- broader civic participation,
- external marketplace activity,
- public-benefit openness,
- provider ecosystem expansion,
- and an implementation sequence not yet centered on a sealed AGI custodian.

## Retrofit Across Existing Core Layers

- `C3` coordination
- `C5` verification
- `C6` knowledge metabolism
- `C7` orchestration
- `C8` settlement

Reason:
These specs likely remain foundational, but they need policy overlays for:

- internal-only novelty workflows,
- release-classed invention artifacts,
- embargoed self-improvement outputs,
- and differentiated externalization pathways.

## Historical / Compatibility Only

- `C4` ASV
- `C43` MCP bridge
- `C46` A2A bridge

Reason:
Under the proposed redesign, these should not define the end-state
architecture.
They remain migration or historical surfaces only.
