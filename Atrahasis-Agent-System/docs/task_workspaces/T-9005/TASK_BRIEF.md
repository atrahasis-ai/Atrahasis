# T-9005 Task Brief

## Operator Prompt Distilled

The operator proposes a major refounding of Atrahasis:
- Atrahasis exists to create mini AGI, Master AGI, and recursive successors.
- The strongest capability layers should remain private and tightly controlled.
- Public and business offerings should expose only bounded derivatives, not raw
  novelty generation or unrestricted agent orchestration.
- Atrahasis should monetize through products, services, and selective licensing
  while retaining technologies required for Atrahasis self-improvement.
- External frontier models may be used during bootstrap, but the long-term
  direction should be an Atrahasis-native model stack aligned to `AASL` and
  `AACP`.

## Analysis Goal

Produce Anu's independent `AAS4` swarm output, grounded in the current
governance, economics, runtime, and compiler specifications, and identify:
- what AGI should mean inside Atrahasis,
- what the consequences of this pivot are,
- which parts of the current system remain valid,
- which parts must be redesigned,
- and which strategic path Anu recommends.

## Success Condition

The result should be decision-grade:
- comparable to other active terminal outputs,
- explicit about contradictions with current specs,
- explicit about safety and governance consequences,
- and clear enough to drive a later system-wide retrofit review.
