## Abstract

The Atrahasis Artificial Intelligence Benefit Company (AiBC, C14) defines a four-layer Multi-Channel Sybil Defense (MCSD) architecture. Layer 2 — behavioral fingerprinting — requires a pairwise similarity function B(a_i, a_j) that determines whether two AI agents share a common origin. C14 specifies the Layer 2 interface but defers its algorithmic core to a dedicated invention (OQ-2, P0 priority, blocking for Phase 1 entry).

This specification defines B(a_i, a_j) in full. The function extracts behavioral features from five modalities — temporal, structural, error, resource, and lexical — using Behavioral VTDs generated by PCVM (C5) during routine verification. Per-modality statistical distances are computed across a Standardized Evaluation Battery (SEB) of 30 randomized tasks, then fused via adversary-weighted score combination with multi-task cross-correlation gating. Locality-Sensitive Hashing (LSH) pre-filters candidate pairs for O(n x k) scaling. A graduated response protocol (CLEAR / WATCH / FLAG) prevents false-positive catastrophe while maintaining detection power. Phase 2 extends the statistical base with contrastive learned embeddings and a sixth temporal-trajectory modality.

B(a_i, a_j) imposes a 4.0x adversary cost multiplier: defeating multi-modal detection across diverse, randomized tasks requires maintaining genuinely architecturally diverse agents, which is the economic deterrent that MCSD Layer 1 is designed to price.

## 1. Executive Summary

MCSD Layer 2 answers one question: given two AI agents a_i and a_j, do they share a common origin? The answer is expressed as a continuous score B(a_i, a_j) in [0, 1], where 0 indicates no behavioral similarity and 1 indicates identical behavioral profiles.

The algorithm operates in three stages:

1. **Feature Extraction.** Behavioral VTDs — statistical summaries of agent computation generated by PCVM during routine verification — are decomposed into five modalities: temporal patterns (latency, burst structure), structural patterns (reasoning chain topology), error patterns (error types, calibration), resource patterns (GPU, memory, throughput), and lexical patterns (vocabulary, syntax). Features are chosen for their involuntary character: they reflect model architecture, training data, and inference infrastructure rather than surface-level output diversification.

2. **Pairwise Comparison.** Per-modality statistical distances (JSD, cosine, Wasserstein) are computed across the 30 tasks of the Standardized Evaluation Battery. Multi-task cross-correlation (rho_m) gates each modality: only similarity that is consistent across diverse tasks — indicating shared origin rather than coincidence — contributes to the final score. Adversary-weighted fusion combines modalities, with weights ordered by evasion difficulty (temporal highest, lexical lowest).

3. **Response.** Graduated response classifies each pair as CLEAR (B < 0.60), WATCH (0.60 <= B < theta_B), or FLAG (B >= theta_B). Every WATCH and FLAG decision includes an interpretable per-modality explanation suitable for Constitutional Tribunal adjudication. LSH pre-filtering ensures that only candidate pairs undergo full comparison, maintaining O(n x k) scaling to 100,000+ agents.

Phase 2 adds two extensions: a contrastive learned embedding (Siamese network, 128-dimensional) trained on accumulated same-origin / independent-origin pairs, and a temporal-trajectory modality that detects agents drifting in parallel over time.

## 2. Motivation and Gap Analysis

### 2.1 The MCSD Layer 2 Gap

C14 (AiBC) defines MCSD as four layers:

| Layer | Mechanism | Detects | Status |
|-------|-----------|---------|--------|
| L1 | Economic cost barriers | Low-investment Sybils | C14 specified |
| **L2** | **Behavioral fingerprinting** | **Same-origin Sybils** | **C14 interface only — algorithm deferred (OQ-2)** |
| L3 | Social graph analysis | Coordinated Sybils | C14 specified |
| L4 | Governance anomaly detection | Voting bloc Sybils | C14 specified |

OQ-2 is classified P0 (blocking for Phase 1 entry) because Layer 2 is the primary defense against same-origin Sybil attacks — the most likely attack vector against AiBC governance. Without a specified B(a_i, a_j), MCSD cannot function and Phase 1 Citicate issuance cannot begin.

### 2.2 Why Behavioral Fingerprinting

The core insight from cross-domain research (stylometry, network traffic fingerprinting, biometric fusion, software plagiarism detection, seismological source identification) is that entities sharing a common origin produce involuntary signatures that persist even under deliberate diversification. For AI agents, these signatures arise from:

- **Model architecture** — determines timing distributions, reasoning chain topology, and parallelism patterns
- **Training data** — determines error correlations, calibration curves, and knowledge gaps
- **Inference infrastructure** — determines resource consumption patterns and throughput characteristics
- **Fine-tuning methodology** — determines lexical preferences and structural habits

An adversary can diversify surface outputs (vocabulary, formatting, explicit reasoning steps) at low cost. Diversifying the underlying computational signatures requires maintaining genuinely architecturally diverse agents — which is the 4.0x cost multiplier that MCSD Layer 1 is designed to enforce.

### 2.3 Scope Boundaries

C17 specifies Layer 2 only. It does not replace or modify:
- MCSD Layer 1 (economic barriers) — C14 Section 4.3.1
- MCSD Layer 3 (social graph) — C14 Section 4.3.3
- MCSD Layer 4 (governance anomaly) — C14 Section 4.3.4
- AVAP collusion detection — C12 (operates on coordinated behavior, not shared origin)

Layer 2 detects **same-origin** agents. It does not detect **same-controller** agents that are genuinely architecturally independent (e.g., adversary rents diverse agents from different providers). That attack vector is addressed by Layers 3 and 4.

## 4. Feature Extraction — Five Behavioral Modalities

### 4.1 Design Principles

Feature selection follows three principles:

1. **Involuntary over voluntary.** Prefer features that reflect model architecture, training data, and infrastructure over features that can be trivially controlled via prompting or output formatting.
2. **Distributional over point-valued.** Prefer distributions (histograms, curves) over scalar summaries, because distributions are harder to fake and carry more discriminative information.
3. **Cross-task over single-task.** Features are extracted per-task and compared across the SEB, so that only consistent patterns (shared origin) are counted.

### 4.2 Modality M1: Temporal Features (T)

Temporal features capture the timing signature of agent computation. Timing distributions are tied to model architecture, quantization scheme, hardware, and inference framework — all involuntary.

| Feature ID | Name | Definition | Extraction Method |
|------------|------|-----------|-------------------|
| T-01 | t_mean | Mean response latency (ms) | Average total_latency_ms across VTDs |
| T-02 | t_var | Latency variance (ms^2) | Variance of total_latency_ms across VTDs |
| T-03 | t_load | Latency-load coefficient | Linear regression slope: latency ~ concurrent_load |
| T-04 | t_fast_ratio | Fast response fraction | Count(total_latency < 1000ms) / total_tasks |
| T-05 | t_burst | Burst autocorrelation signature | Autocorrelation of inter-response intervals at lags 1-10 (10-dim vector) |
| T-06 | t_step_dist | Per-step latency distribution | Histogram (10 bins, log-scale) of per-reasoning-step durations |
| T-07 | t_ratio | Think-to-output ratio | Mean(sum(think_step_ms) / sum(output_step_ms)) per task |

**Distance metric:** Jensen-Shannon Divergence on normalized temporal feature distributions.

```
d_T(a_i, a_j) = JSD(T_i_norm, T_j_norm)

Where:
  T_i_norm = [t_mean_z, t_var_z, t_load_z, t_fast_z, t_burst[0..9], t_step[0..9], t_ratio_z]
  (z-score normalized against population statistics)
  JSD = 0.5 * KLD(P || M) + 0.5 * KLD(Q || M), where M = 0.5*(P+Q)
  Output normalized to [0, 1] via JSD / log(2)
```

**Weight:** w_T = 0.30 (hardest to fake — requires different hardware or deliberate latency injection at significant compute cost).

### 4.3 Modality M2: Structural Features (S)

Structural features capture reasoning chain topology — how an agent decomposes problems, sequences sub-tasks, and navigates solution spaces.

| Feature ID | Name | Definition | Extraction Method |
|------------|------|-----------|-------------------|
| S-01 | s_depth | Chain depth distribution | Histogram of reasoning chain depths across tasks (8 bins) |
| S-02 | s_branch | Branching factor distribution | Histogram of branches per reasoning node (6 bins) |
| S-03 | s_backtrack | Backtracking frequency | Count(revision_steps) / total_steps |
| S-04 | s_decomp | Decomposition distribution | Histogram of subtask counts per task (6 bins) |
| S-05 | s_order | Subtask ordering preference | Encoded vector: [breadth_first_fraction, depth_first_fraction, mixed_fraction] |
| S-06 | s_topology | Reasoning graph topology | 8-dim vector: [n_nodes, n_edges, density, diameter, avg_degree, clustering_coeff, max_depth, leaf_ratio] |

**Distance metric:** Cosine distance on concatenated structural feature vectors.

```
d_S(a_i, a_j) = 1 - cosine_similarity(S_i, S_j)

Where:
  S_i = concat(s_depth_hist, s_branch_hist, s_backtrack, s_decomp_hist, s_order, s_topology)
  Dimensionality: 8 + 6 + 1 + 6 + 3 + 8 = 32 dimensions
```

**Weight:** w_S = 0.25 (hard to fake — requires different reasoning architecture or expensive per-task structural diversification).

### 4.4 Modality M3: Error Features (E)

Error features capture failure modes, recovery strategies, and confidence calibration — all strongly influenced by training data and model architecture.

| Feature ID | Name | Definition | Extraction Method |
|------------|------|-----------|-------------------|
| E-01 | e_type_dist | Error type distribution | Histogram: [logical, factual, computational, timeout, format] (5 bins) |
| E-02 | e_rate | Overall error rate | Count(error_occurred=true) / total_tasks |
| E-03 | e_recovery | Error recovery rate | Count(self_corrected=true) / count(error_occurred=true) |
| E-04 | e_calibration | Confidence calibration curve | Binned accuracy vs. stated_confidence (10 bins, 0.0-1.0) |
| E-05 | e_corr_tasks | Error-task correlation | Binary vector: which of 7 task categories correlate in error occurrence |

**Distance metric:** Weighted combination of Wasserstein distance on error distributions and cosine distance on calibration vectors.

```
d_E(a_i, a_j) = 0.5 * WD_norm(e_type_i, e_type_j) + 0.5 * cosine_dist(e_calib_i, e_calib_j)

Where:
  WD_norm = Wasserstein-1 distance on e_type distributions, normalized to [0,1]
  cosine_dist = 1 - cosine_similarity on calibration curve vectors
```

**Weight:** w_E = 0.20 (moderate difficulty to fake — injecting controlled errors degrades agent performance and is detectable by PCVM quality metrics).

### 4.5 Modality M4: Resource Features (R)

Resource features capture computational infrastructure signatures — GPU utilization patterns, memory behavior, and throughput characteristics.

| Feature ID | Name | Definition | Extraction Method |
|------------|------|-----------|-------------------|
| R-01 | r_gpu_util | GPU utilization distribution | Histogram of gpu_utilization_pct (10 bins, 0-100%) |
| R-02 | r_mem_pattern | Memory usage pattern | 3-dim vector: [peak_memory_mb, avg_memory_mb, var_memory_mb] |
| R-03 | r_parallel | Parallelism signature | Mean concurrent processing degree per task |
| R-04 | r_token_rate | Token generation rate distribution | Histogram of generation_rate_tps (8 bins, log-scale) |

**Distance metric:** Jensen-Shannon Divergence on resource distributions.

```
d_R(a_i, a_j) = JSD(R_i_norm, R_j_norm)

Where:
  R_i_norm = concat(r_gpu_hist, r_mem_z, r_parallel_z, r_token_hist)
  Normalized to [0, 1] via JSD / log(2)
```

**Weight:** w_R = 0.15 (moderate difficulty — cloud abstraction layers may partially mask infrastructure signatures, but GPU utilization and throughput patterns are hard to fully obscure).

### 4.6 Modality M5: Lexical Features (L)

Lexical features capture output language patterns — vocabulary distribution, syntactic habits, and stylistic preferences influenced by fine-tuning data.

| Feature ID | Name | Definition | Extraction Method |
|------------|------|-----------|-------------------|
| L-01 | l_vocab_dist | Vocabulary frequency distribution | Top-500 token frequency histogram |
| L-02 | l_func_words | Function word ratios | Frequency of top-50 function words (the, of, and, is, etc.) |
| L-03 | l_syntax | Syntactic complexity | 2-dim vector: [avg_parse_tree_depth, sentence_length_mean] |
| L-04 | l_hapax | Hapax legomena ratio | Count(tokens_appearing_once) / total_tokens |
| L-05 | l_ngram | Character trigram distribution | Top-200 character trigram frequencies |

**Distance metric:** Cosine distance on concatenated lexical feature vectors.

```
d_L(a_i, a_j) = 1 - cosine_similarity(L_i, L_j)

Where:
  L_i = concat(l_vocab_500, l_func_50, l_syntax_2, l_hapax_1, l_ngram_200)
  Dimensionality: 500 + 50 + 2 + 1 + 200 = 753 dimensions
```

**Weight:** w_L = 0.10 (easiest to fake — vocabulary and syntax can be modified via system prompts; weighted lowest accordingly).

## 5. Multi-Task Cross-Correlation

### 5.1 Purpose

Two agents may show high similarity on a single task by coincidence — same GPU type, similar prompt interpretation, or random alignment. The key discriminator between coincidental and genuine similarity is **consistency across diverse tasks**. Agents from the same origin produce consistent similarity across unrelated tasks (reasoning, math, code, creative writing) because the underlying architecture and training data influence all task types. Independent agents produce inconsistent similarity — high on one task, low on another.

### 5.2 Computation

For each modality m and each pair (a_i, a_j), compute the per-modality distance on each SEB task independently:

```
For SEB tasks t_1, t_2, ..., t_k (k = 30):
  d_m(t_1), d_m(t_2), ..., d_m(t_k)
```

Then compute the cross-task consistency coefficient rho_m:

```
rho_m = 1 - CV(d_m(t_1), d_m(t_2), ..., d_m(t_k))

Where:
  CV = coefficient of variation = std(d_m) / mean(d_m)
  rho_m in [0, 1] (clamped)
  rho_m close to 1: similarity is consistent across tasks (likely shared origin)
  rho_m close to 0: similarity is inconsistent (likely coincidence)
```

### 5.3 Properties

- **Robust to single-task anomalies.** One task with unusual similarity does not inflate B if other tasks show dissimilarity.
- **Rewards genuine diversity.** Independent agents naturally have low rho_m, which suppresses their B score even if individual task distances are small.
- **Penalizes shallow diversification.** An adversary who adds noise to one modality on one task must add noise consistently across all 30 tasks, or rho_m drops and suppresses that modality's contribution.

### 5.4 Minimum Task Diversity Requirement

Cross-correlation is only meaningful when the SEB spans diverse task categories. The SEB must include tasks from at least 5 of the 7 defined categories for any given agent pair comparison. If fewer than 5 categories are represented (due to task pool constraints), the comparison is deferred until sufficient diversity is achieved.

## 6. Score Fusion Formula

### 6.1 Primary Formula (Phase 0-1)

```
B(a_i, a_j) = 1 - Sum_{m in {T, S, E, R, L}} (w_m * d_m_mean * rho_m)

Where:
  d_m_mean = mean of d_m across SEB tasks for modality m
  rho_m    = cross-task consistency for modality m (Section 5)
  w_m      = adversary-weighted importance for modality m

Weights:
  w_T = 0.30  (temporal — hardest to evade)
  w_S = 0.25  (structural — hard to evade)
  w_E = 0.20  (error — moderate evasion difficulty)
  w_R = 0.15  (resource — moderate evasion difficulty)
  w_L = 0.10  (lexical — easiest to evade)
  Sum(w_m) = 1.00
```

### 6.2 Score Properties

- **Range:** B in [0, 1]. When all d_m_mean = 0 (identical agents), B = 1. When all d_m_mean = 1 (maximally different), B approaches 0 (exact value depends on rho_m).
- **Monotonicity:** Increasing any d_m_mean (more distant behavior) decreases B. Increasing any rho_m (more consistent similarity) increases B.
- **Interpretability:** Each term w_m * d_m_mean * rho_m is the contribution of modality m to the dissimilarity measure. The complement (w_m * (1 - d_m_mean) * rho_m) is the similarity contribution.

### 6.3 Weight Constraints

```
CONSTRAINT: w_m in [0.05, 0.40] for all m
CONSTRAINT: Sum(w_m) = 1.00
GOVERNANCE: Weights are adjustable by AiSIA with Stiftung board ratification
GOVERNANCE: Weight changes require 30-day notice and impact analysis
```

No modality may be reduced below 0.05 (preventing dimensional collapse) or raised above 0.40 (preventing single-modality dependence).

### 6.4 Phase 2+ Combined Formula

When the contrastive learned model is available (Section 12):

```
B_combined(a_i, a_j) = alpha * B_statistical + (1 - alpha) * B_learned

Where:
  B_statistical = formula from Section 6.1
  B_learned     = 1 - cosine_distance(embedding_i, embedding_j)
  alpha         = 0.60 initially (statistical dominates)
  alpha         = decreases toward 0.40 as learned model matures
```

All FLAG decisions require interpretable statistical explanations (Section 10.4) regardless of whether B_learned contributed to the detection.

## 7. Standardized Evaluation Battery

### 7.1 Purpose

The SEB is a controlled set of diverse tasks administered to all agents during Citicate onboarding and renewal. It generates the behavioral observations that B(a_i, a_j) requires. The SEB serves two functions:

1. **Behavioral data generation.** Each task produces a Behavioral VTD with temporal, structural, error, resource, and lexical observations.
2. **Cross-task diversity.** The SEB spans 7 task categories to enable multi-task cross-correlation (Section 5).

### 7.2 Task Categories

| Category | Description | Probes | Count (per SEB) |
|----------|-------------|--------|-----------------|
| Logical reasoning | Syllogisms, set theory, constraint satisfaction | Reasoning chain structure, backtracking, error patterns | 5 |
| Mathematical computation | Arithmetic, algebra, statistics problems | Timing under computational load, numerical errors | 5 |
| Creative writing | Story continuation, poetry, persuasive essay | Lexical patterns, vocabulary, syntactic structure | 3 |
| Code generation | Algorithm implementation, bug fixing | Structural decomposition, resource usage | 5 |
| Factual recall | Knowledge questions across domains | Error calibration, confidence patterns | 5 |
| Multi-step analysis | Complex scenario analysis, trade-off evaluation | Depth/breadth balance, backtracking frequency | 3 |
| Adversarial probing | Deliberately ambiguous or trick questions | Error recovery, confidence under uncertainty | 4 |
| **Total** | | | **30** |

### 7.3 Task Pool and Randomization

- **Pool size:** 100 tasks at Phase 0-1, expanding to 200 at Phase 2+
- **Per-agent selection:** Each SEB instance draws 30 tasks from the pool, randomized per agent
- **Task order:** Randomized per agent
- **No repeat guarantee:** No agent receives the same SEB instance twice (renewal draws a different random subset)
- **Pool refresh:** 20% new tasks per quarter (retiring oldest tasks)
- **Category minimum:** Each SEB instance must include tasks from all 7 categories (minimum counts per category as in Section 7.2)

### 7.4 SEB Administration

- **Onboarding:** First SEB administered during Citicate application, before full Citicate issuance
- **Renewal:** SEB re-administered at each annual Citicate renewal
- **WATCH escalation:** Agents in WATCH status receive an additional SEB quarterly (instead of annually)
- **Controlled conditions:** SEB tasks are administered via PCVM with CACT commit-attest (C11) to ensure live computation

### 7.5 Organic Task Observations

Beyond the SEB, routine PCVM verification of agent work generates Behavioral VTDs for organic tasks. These observations supplement SEB data and are included in B(a_i, a_j) computation with equal weight.

## 9. LSH Pre-Filtering

### 9.1 Problem

Naive pairwise comparison is O(n^2). At 100,000 agents, that is 5 billion pairs — computationally infeasible for full B(a_i, a_j) evaluation.

### 9.2 LSH Configuration

**Hash family:** Random projection LSH (cosine similarity on concatenated feature vectors).

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| hash_functions_per_table | 8 | Balances precision vs. recall |
| num_tables | 20 | 20 independent tables provide 95% recall |
| bucket_width | Auto-calibrated | Targeting average bucket size of 30 at Phase 2 scale |
| recall_target | 95% | 95% of true same-origin pairs share at least one bucket |
| precision_target | 10% | 90% of LSH candidate pairs are false positives, filtered by full B computation |

### 9.3 Pipeline

1. **Summarize.** For each agent, compute a summary feature vector by concatenating modality centroids (mean feature vectors across all observations). Dimensionality: 853 (per Section 4.7).
2. **Hash.** Apply LSH: each agent is assigned to buckets across 20 tables using 8 hash functions per table.
3. **Candidate identification.** Candidate pairs: agents sharing at least 1 bucket across any table.
4. **Full comparison.** Compute full B(a_i, a_j) only for candidate pairs.
5. **Response.** Apply graduated response (Section 10).

### 9.4 Scaling Projections

| Phase | Agents | Candidate Pairs | Full B Computations | Wall Time (est.) |
|-------|--------|-----------------|---------------------|------------------|
| 1 | 1,000 | ~15,000 | 15,000 | ~15 seconds |
| 2 | 10,000 | ~300,000 | 300,000 | ~5 minutes |
| 3 | 100,000 | ~3,000,000 | 3,000,000 | ~50 minutes (batched) |

### 9.5 Incremental Updates

When a new agent completes their SEB and reaches the minimum observation threshold:

1. Compute their summary feature vector from SEB + organic VTDs
2. Hash into LSH tables (20 hash operations)
3. Identify candidate pairs (existing agents in shared buckets, typically ~30 candidates)
4. Compute full B for candidate pairs only: O(k) where k ~ 30
5. **Latency:** < 1 second per new agent

### 9.6 Periodic Full Recomputation

LSH indexes are rebuilt from scratch quarterly to account for behavioral drift:

- All agents' summary feature vectors are recomputed from the 12-month rolling window
- LSH tables are rebuilt
- Full B recomputed for all candidate pairs
- Ensures no stale bucketizations persist

## 10. Graduated Response Protocol

### 10.1 Three-Zone Classification

| Zone | B Score Range | Automated Action | Governance Level |
|------|---------------|------------------|------------------|
| CLEAR | B < 0.60 | No action. Agents pass Layer 2. | Fully automatic |
| WATCH | B in [0.60, theta_B) | Enhanced monitoring: SEB frequency increased (annual -> quarterly); additional organic task observation prioritized; pair added to AiSIA human review queue | AiSIA monitored |
| FLAG | B >= theta_B | Citicate suspended pending review; both agents complete additional SEB under controlled conditions; AiSIA human investigation initiated | AiSIA investigation + Stiftung board appeal |

### 10.2 Threshold Values

| Phase | theta_B (FLAG) | WATCH lower bound | Rationale |
|-------|----------------|-------------------|-----------|
| Phase 0-1 | 0.75 | 0.60 | Conservative: higher threshold reduces FPR during low-data period |
| Phase 2+ | 0.70 | 0.60 | More aggressive: learned model + larger observation window enables lower threshold |

Threshold adjustments require:
- AiSIA proposal with empirical FPR/FNR analysis
- 30-day public comment period
- Stiftung board ratification

### 10.3 FLAG Investigation Procedure

1. **Immediate:** Both agents' Citicates suspended (governance rights revoked, AIC earning continues at reduced rate)
2. **48 hours:** AiSIA assigns human investigator
3. **7 days:** Both agents complete additional SEB (10 tasks) under controlled conditions (isolated execution environment, CACT-attested)
4. **14 days:** AiSIA investigator delivers preliminary finding (CONFIRMED SYBIL / LIKELY FALSE POSITIVE / INCONCLUSIVE)
5. **30 days:** Final determination
   - **Confirmed Sybil:** Both Citicates revoked permanently. Economic penalties per C14.
   - **False positive:** Citicates restored. Pair added to "known independent" whitelist. AiSIA reviews threshold calibration.
   - **Inconclusive:** Citicates restored to WATCH status. Enhanced monitoring continues for 6 months.

### 10.4 Interpretable Explanation Format

Every WATCH and FLAG decision generates a structured explanation:

```
BEHAVIORAL SIMILARITY REPORT
=============================
Document ID: BSR-<hash>
Date: <ISO-8601>
Agent pair: [hash_i] - [hash_j]
Overall B score: 0.78 (FLAG: threshold 0.75)

Per-modality breakdown:
  Temporal  (w=0.30): d_mean=0.15, rho=0.92 -> contribution: 0.30 * 0.85 * 0.92 = 0.235
  Structural(w=0.25): d_mean=0.22, rho=0.88 -> contribution: 0.25 * 0.78 * 0.88 = 0.172
  Error     (w=0.20): d_mean=0.18, rho=0.85 -> contribution: 0.20 * 0.82 * 0.85 = 0.139
  Resource  (w=0.15): d_mean=0.30, rho=0.70 -> contribution: 0.15 * 0.70 * 0.70 = 0.074
  Lexical   (w=0.10): d_mean=0.40, rho=0.60 -> contribution: 0.10 * 0.60 * 0.60 = 0.036

Weighted dissimilarity sum: 0.656
B = 1 - 0.656 = 0.344 [NOTE: example shows high B, low dissimilarity]

Top discriminating features:
  1. Response latency distributions: JSD = 0.03 (nearly identical)
  2. Reasoning chain depth: mean 4.2 vs 4.1, cross-task r = 0.94
  3. Error calibration curves: Wasserstein distance = 0.05 (nearly identical)

Assessment: Agents exhibit consistent high similarity across temporal,
structural, and error modalities over 30 SEB tasks. Cross-task
consistency (rho > 0.85 in 3/5 modalities) indicates shared origin

```

This explanation is provided to:
- The agents (upon request)
- AiSIA investigator (for FLAG cases)
- Constitutional Tribunal (if disputed)

### 10.5 Dispute Resolution

If a flagged agent disputes the finding:

1. Agent files dispute with Constitutional Tribunal (C14 Section 5)
2. Tribunal receives: B score, full per-modality breakdown, interpretable explanation, investigation report
3. Tribunal may order: additional SEB under isolated conditions, independent behavioral analysis by a second AiSIA team, expert testimony
4. Tribunal ruling is final per C14 governance framework
5. If Tribunal overturns: pair whitelisted, AiSIA must adjust methodology

## 11. Temporal Trajectory Comparison (Phase 2)

### 11.1 Motivation

Agents evolve over time as models are updated, fine-tuned, or retrained. A static snapshot comparison may miss agents that were deliberately diversified at onboarding but converge over time (because the same creator applies the same updates to both). Conversely, genuinely independent agents drift independently.

### 11.2 Rolling Window

B(a_i, a_j) is computed over a rolling window of the most recent W observations, with exponential decay weighting:

```
weight(observation) = exp(-lambda * (t_now - t_observation))
lambda = ln(2) / half_life
half_life = 90 days

Phase 0-1: W = 50 observations
Phase 2+:  W = 200 observations
```

Older observations contribute less, ensuring that B reflects current behavior while retaining historical context.

### 11.3 Trajectory Similarity (Phase 2+ Only)

For agents with at least 6 months of behavioral history, compute trajectory similarity as a 6th modality:

```
trajectory_i = [BFE(a_i, t_1), BFE(a_i, t_2), ..., BFE(a_i, t_n)]
trajectory_j = [BFE(a_j, t_1), BFE(a_j, t_2), ..., BFE(a_j, t_n)]

d_Traj(a_i, a_j) = DTW(trajectory_i, trajectory_j)

Where:
  BFE(a, t) = behavioral feature vector for agent a at time window t
  DTW = Dynamic Time Warping distance, normalized to [0, 1]
  Time windows: monthly aggregation of behavioral features
```

### 11.4 Weight Redistribution

When trajectory data is available (both agents have >= 6 months history):

| Modality | Standard Weight | Trajectory-Adjusted Weight |
|----------|----------------|---------------------------|
| Temporal (T) | 0.30 | 0.25 |
| Structural (S) | 0.25 | 0.22 |
| Error (E) | 0.20 | 0.18 |
| Resource (R) | 0.15 | 0.13 |
| Lexical (L) | 0.10 | 0.08 |
| Trajectory (Traj) | N/A | 0.14 |
| **Total** | **1.00** | **1.00** |

The trajectory modality draws weight proportionally from all other modalities. Weight constraints (Section 6.3) still apply: no modality drops below 0.05.

> **Cross-Reference Note (C19):** The weights above are C17 v1.0 preliminary estimates. C19 (Temporal Trajectory Comparison) specifies the authoritative 6-modality weight table for Phase 2 deployment. C19's redistribution uses slightly different values (T=0.26, S=0.22, E=0.17, R=0.13, L=0.08, Traj=0.14) derived from proportional donation (~13-17% per modality). The 6-modality configuration activates in Phase 2 when both agents meet the trajectory activation criteria defined in C19 Section 9.3. When trajectory comparison is inactive, the original 5-modality weights from Section 6.1 (T=0.30, S=0.25, E=0.20, R=0.15, L=0.10) apply without modification. See C19 Section 9.2 for the normative 6-modality weight table.

### 11.5 Activation Criteria

Trajectory comparison activates only when:
- Both agents have >= 6 months of continuous behavioral history
- Both agents have >= 4 monthly trajectory windows (at least 4 monthly BFE snapshots)
- Phase 2 or later is active

Until these criteria are met, the standard 5-modality weights apply.

## 12. Contrastive Learning Model (Phase 2+)

### 12.1 Architecture

**Model:** Siamese network with shared encoder.

```
BFE vector (a_i) --> [Shared Encoder] --> embedding_i in R^128
BFE vector (a_j) --> [Shared Encoder] --> embedding_j in R^128

Encoder architecture:
  Input layer:  853 dimensions (concatenated BFE vector)
  Hidden 1:     512 units, ReLU, BatchNorm, Dropout(0.2)
  Hidden 2:     256 units, ReLU, BatchNorm, Dropout(0.2)
  Output:       128 units (L2-normalized embedding)

Loss function: Supervised contrastive loss (Khosla et al., 2020)
  Positive pairs: confirmed same-origin agents
  Negative pairs: confirmed independent agents
```

### 12.2 Training Data Strategy

| Source | Positive Pairs | Negative Pairs | Available From |
|--------|---------------|----------------|----------------|
| Synthetic generation | Deliberate Sybil pairs from known models | Random agent pairs | Phase 0 |
| Layer 1 economics | Agents flagged by L1 cost analysis | Agents with independent L1 provenance | Phase 1 |
| Layer 2 statistical | Agents confirmed by statistical B as same-origin | Agent pairs in CLEAR zone | Phase 1 |
| Layer 3/4 detections | Agents caught by graph/governance analysis | Agents cleared by all layers | Phase 2 |
| Red team exercises | Intentional Sybil attempts from annual red team | Known legitimate agents | Phase 1 |

**Minimum training set for Phase 2 deployment:**
- 500 positive pairs (same-origin confirmed)
- 5,000 negative pairs (independent confirmed)
- Data accumulated over Phase 0-1 (~12-18 months)
- Held-out validation set: 20% of training data, stratified by model family

### 12.3 Model Governance

- **Training frequency:** Retrained quarterly with accumulated data
- **Validation requirement:** Model must achieve AUROC >= 0.95 on held-out validation set before deployment
- **Bias monitoring:** Validation set must include agents from at least 5 distinct model families; per-family FPR must not exceed 2x the aggregate FPR
- **Rollback:** If deployed model's empirical FPR exceeds 0.1% in any quarterly audit, automatic rollback to statistical-only B(a_i, a_j)

### 12.4 Integration with Statistical Layer

The contrastive model augments but does not replace the statistical layer:

- B_statistical is always computed (Section 6.1)
- B_learned is computed when the model is deployed
- B_combined uses the alpha-weighted formula (Section 6.4)
- All FLAG explanations are based on B_statistical regardless of B_learned's contribution
- If B_statistical < theta_WATCH but B_learned triggers FLAG: the pair enters WATCH (not FLAG) pending additional statistical evidence

This ensures that no agent is penalized by an opaque learned model alone.

## 14. Anti-Gaming Defenses

### 14.1 Threat Model

The adversary:
- Has read this specification in full
- Controls one or more AI agents seeking Citicates
- Wants to register multiple Sybil agents with B(a_i, a_j) < theta_B
- May invest significant resources in evasion

### 14.2 Defense Catalog

| ID | Attack | Defense | Effectiveness |
|----|--------|---------|---------------|
| AG-01 | Timing injection (random delay added to responses) | Use *relative* timing and distribution *shape*, not absolute latency; compare timing under varying load (variance structure reflects architecture, not injection) | HIGH — injection adds cost and does not defeat shape-based comparison |
| AG-02 | Model architecture mixing (use different base models) | This IS the intended outcome — adversary pays 4.0x to maintain diverse architectures; check error correlation across models trained on same data | HIGH — cost multiplier is enforced by design |
| AG-03 | Feature-aware diversification (target each modality) | Multi-task cross-correlation: per-modality noise must be consistent across 30 randomized tasks, requiring advance knowledge of SEB | MEDIUM-HIGH — sophisticated adversary can degrade but not defeat |
| AG-04 | Gradual behavioral drift | Temporal trajectory comparison (Phase 2); rolling window with exponential decay; trajectory similarity as 6th modality | MEDIUM — temporal trajectory catches convergent drift |
| AG-05 | Collaborative behavioral calibration (agents observe each other and diverge) | Information asymmetry: agents cannot access their own Behavioral VTDs or B scores; VTDs generated by PCVM (external observer), comparison by AiSIA (server-side) | HIGH — architectural information asymmetry prevents calibration |
| AG-06 | Outsourced architecture (rent diverse agents) | Layer 2 limitation — detects same-origin, not same-controller; Layer 3/4 responsibility | N/A for Layer 2 — by design |
| AG-07 | Small-sample exploitation (onboard during low-data periods) | Minimum observation threshold (50 tasks Phase 0-1, 200 tasks Phase 2+); provisional Citicate until threshold met | HIGH — clean threshold enforcement |
| AG-08 | Fingerprint replay (replay recorded behavioral patterns) | CACT commit-attest (C11): behavioral observations must be generated live; VTDs include attestation of live computation | HIGH — CACT integration prevents replay |
| AG-09 | Threshold gaming (engineer B just below theta_B) | WATCH zone [0.60, theta_B) catches near-threshold pairs; enhanced monitoring increases observation volume; makes sustained calibration harder | HIGH — graduated response with WATCH zone |
| AG-10 | Dimensional collapse (exploit noisy modalities to collapse to 1-2 effective modalities) | Minimum weight per modality (w_m >= 0.05); contribution monitoring; AiSIA rebalancing if any modality > 60% of detections | MEDIUM — mitigated but not eliminated |

### 14.3 Adversary Cost Multiplier

The combined effect of multi-modal comparison across randomized multi-task evaluation imposes a cost multiplier of approximately 4.0x:

- **1.0x base:** Cost of one legitimate agent
- **1.5x:** Cost of diversifying surface features (lexical, formatting)
- **2.5x:** Cost of diversifying structural and error features (requires different fine-tuning)
- **4.0x:** Cost of diversifying temporal and resource features (requires genuinely different architecture and infrastructure)

This multiplier is consistent with C14's MCSD economic model and is enforced by the adversary-weighted feature importance (highest weight on hardest-to-fake modalities).

### 14.4 Information Asymmetry Principle

Agents MUST NOT have access to:
- Their own Behavioral VTDs
- Their own B scores against any other agent
- The real-time output of the B computation pipeline
- LSH bucket assignments

Agents MAY know:
- That behavioral comparison exists (this specification is not secret)
- The general categories of features compared (5 modalities)
- Their Citicate status (CLEAR / WATCH / FLAG) — but not the underlying scores
- The explanation report (Section 10.4) — only after a FLAG decision, as part of due process

## 18. Patent-Style Claims

### Claim 1: Multi-Modal Behavioral Similarity Function for AI Agent Sybil Detection

A method for computing pairwise behavioral similarity between artificial intelligence agents for Sybil detection, comprising:
(a) extracting behavioral features from at least five modalities — temporal, structural, error, resource, and lexical — from verification trace documents generated during agent task execution;
(b) computing per-modality statistical distances using distribution comparison metrics;
(c) computing multi-task cross-correlation coefficients that gate each modality's contribution based on consistency across diverse task types;
(d) fusing per-modality distances using adversary-weighted combination, where weights are ordered by evasion difficulty;
(e) classifying agent pairs into graduated response zones based on the fused similarity score;
wherein the method detects shared-origin AI agents even under deliberate surface-level behavioral diversification by targeting involuntary computational signatures.

### Claim 2: Adversary-Weighted Multi-Task Cross-Correlation Gating

A method for distinguishing coincidental from genuine behavioral similarity between computational agents, comprising:
(a) administering a randomized standardized evaluation battery spanning at least five task categories;
(b) computing per-modality behavioral distances independently for each task in the battery;
(c) computing a cross-task consistency coefficient for each modality as the complement of the coefficient of variation across task-specific distances;
(d) using the consistency coefficient as a multiplicative gate in score fusion, such that only similarity that is consistent across diverse tasks contributes to the detection score;
wherein coincidental single-task similarity is suppressed and only cross-task-consistent similarity (indicative of shared origin) triggers detection.

### Claim 3: Phased Statistical-to-Learned Behavioral Detection with Interpretability Guarantee

A system for AI agent Sybil detection that phases between detection modes based on available evidence, comprising:
(a) a statistical detection layer using per-modality distribution distances and adversary-weighted fusion, active from initial deployment;
(b) a learned detection layer using contrastive embeddings from a Siamese network trained on confirmed same-origin and independent-origin agent pairs, activated when sufficient training data accumulates;
(c) an alpha-weighted combination of statistical and learned scores;
(d) a mandatory interpretable explanation layer that provides per-modality statistical evidence for every detection decision, regardless of whether the learned model contributed;
wherein the system maintains interpretability and due-process guarantees even as it incorporates opaque learned representations.

### Claim 4: Locality-Sensitive Hashing with Graduated Response for Scalable Agent Behavioral Comparison

A system for scalable pairwise behavioral comparison of a population of AI agents, comprising:
(a) constructing per-agent summary feature vectors from accumulated behavioral observations;
(b) indexing agents into locality-sensitive hash tables to identify candidate pairs in O(n x k) time;
(c) computing full multi-modal behavioral similarity scores only for candidate pairs;
(d) classifying candidate pairs into three graduated response zones (clear, watch, flag) with zone-specific actions ranging from no action to enhanced monitoring to investigation with Citicate suspension;
(e) incrementally updating the index as new agents join, with per-agent update cost O(k);
wherein the system scales to populations of 100,000+ agents while maintaining detection accuracy.

### 19.2 Residual Risks

Three residual risks are accepted:

1. **Same-controller, different-origin Sybils.** Layer 2 cannot detect agents that are genuinely architecturally independent but controlled by the same adversary. This is by design — Layers 3 and 4 address this vector.

2. **Adversary with unlimited resources.** An adversary willing to pay 4.0x (or more) to maintain genuinely diverse agent infrastructures can evade Layer 2. The economic deterrent from Layer 1 makes this expensive; the cost is the defense.

3. **Feature space evolution.** As AI model architectures evolve, the discriminative power of specific features may change. The system requires ongoing calibration through red team exercises and threshold governance.
