# C41 Landscape Report - Layered Semantic Capability Manifest (LSCM)

## Market and ecosystem landscape

The surrounding landscape already contains:
- agent cards and model/plugin manifests,
- API discovery documents,
- security metadata documents,
- registry and marketplace records,
- signed release and artifact manifests.

What the current landscape does not provide is a single discovery artifact that
combines:
- signed endpoint trust posture,
- semantic-capability disclosure,
- protocol message-family support,
- ontology snapshot disclosure,
- and bridge-honest provenance posture.

## Competitive posture

### Strong incumbents
- generic API-description ecosystems,
- AI plugin and agent-card patterns,
- identity-provider metadata flows.

### Open gap

No incumbent pattern appears to be both:
- semantically rich enough for the Alternative B stack,
- and bounded enough to remain a stable discovery contract.

That gap matters because Alternative B must later support:
- registry search and trust scoring,
- A2A bridge translation,
- SDK auto-discovery,
- conformance validation,
- and cross-layer integration.

## Strategic implication

`C41` does not compete by replacing every existing manifest pattern. It creates
the protocol-native source-of-truth document that later registries, bridges, and
tooling can consume without inventing missing trust and capability semantics.
