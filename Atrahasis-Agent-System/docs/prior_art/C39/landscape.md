# C39 Landscape Analysis - LCML

## Current landscape

### A2A
- Strong on agent discovery, task lifecycle, and enterprise governance.
- Weak on semantic payload accountability and message lineage as first-class protocol identity.

### MCP
- Strong on tools, resources, prompts, and sampling.
- Weak on unified multi-agent provenance, canonical semantic identity, and sovereign end-state control.

### Atrahasis legacy AACP lineage
- Strong on fixed classes, semantic bundles, and lineage.
- Incomplete for Alternative B capability families and inconsistent across draft-era inventories.

## Strategic gap

No currently adopted system offers all of the following together:
- one sovereign message inventory across agent discovery and tool connectivity,
- mandatory lineage for every interaction family,
- class-level visibility of native versus bridge provenance,
- bounded growth from a known baseline rather than endless feature accretion.

## Competitive reality

`T-211` does not compete on message classes alone. Message-count expansion is easy. The competitive burden is:
- inventory coherence,
- implementation tractability,
- downstream compatibility with transports, security, and semantics,
- preservation of the Alternative B sovereignty argument.

## Landscape takeaway

The environment rewards a design that is:
- explicit enough to replace MCP/A2A operational surfaces,
- compact enough to remain implementable,
- disciplined enough that downstream specs do not reinterpret class meaning.
