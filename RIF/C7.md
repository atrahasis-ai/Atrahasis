# C7 — CIOS Architecture Review and Reinvention

**Date:** 2026-03-10
**Stage:** IDEATION

## Domain Translator Brief

### Current CIOS State
CIOS (Collective Intelligence Operating System) is scattered across 8+ documents with no single authoritative specification. It is an implied layer referenced by other components but never formally designed.

### Current Functions (5)
1. **Goal decomposition** — high-level objectives → AASL-typed subtask directives
2. **Agent registry & lifecycle** — authoritative roster (AgentId, pubkey, capabilities, stake, reputation)
3. **Locus assignment** — semantic routing of tasks to domains
4. **Configuration governance** — tidal function parameters, epoch lengths, verification thresholds
5. **Settlement coordination** — forwards economic ledgers between C3 and Settlement Plane

### Integration Requirements from C3/C4/C5/C6
CIOS must provide:
- Authoritative agent roster synchronized ±500ms NTP
- Sybil-resistant identity layer
- Clock synchronization for epoch boundaries
- Governance coordination (G-class BFT consensus, ETR)
- Locus-to-agent mapping in real-time
- Economic settlement forwarding
- Cross-component failure detection and graceful degradation
- Configuration versioning with ≥1 epoch lead-time

CIOS must NOT:
- Override PCVM claim classifications (INV-M2)
- Interfere with VRF verifier selection (INV-M3)
- Modify VTDs or MCTs (INV-M7)
- Interrupt EMA metabolic phases (INV-E3)
- Override SHREC allocations
- Manually assign operation classes

### Core Tension
Current CIOS is described as a centralized executive coordinating decentralized autonomous subsystems. Every layer below it (C3, C5, C6) is designed to operate without central direction. What does an orchestration layer actually DO when its subsystems are sovereign?

## Ideation Council Deliberation

### Round 1 — Initial Proposals

**Visionary:** CIOS is a ghost — referenced everywhere, specified nowhere. Its implied centralized executive contradicts the decentralized autonomous subsystems below it. Proposed: Intent Metabolism Engine — translates goals into substrate conditions, doesn't execute directly. Like the hypothalamus, not the prefrontal cortex.

**Systems Thinker:** Formalized 7 irreducible CIOS functions: (1) goal-to-task decomposition, (2) agent identity/registry, (3) locus topology management, (4) clock synchronization, (5) governance coordination, (6) cross-component failure detection, (7) economic settlement routing. Proposed splitting into Infrastructure Plane (boring plumbing) and Executive Plane (real design challenge).

**Critic:** Pushed back — CIOS is underspecified, not necessarily wrong. Centralized executive coordinating autonomous subsystems matches the human nervous system (prefrontal cortex doesn't micromanage heart). Problem is lack of formalization, not the concept itself.

**Adversarial Analyst:** Centralization = bottleneck. Goal decomposition requiring LLM inference at O(goals × inference_cost) becomes throughput ceiling at scale. Can goal decomposition be hierarchical like C3's scheduling?

**Simplification Agent:** Enumerated what's genuinely unowned: (1) agent registry, (2) goal decomposition, (3) governance voting, (4) cross-layer failure detection, (5) settlement forwarding. Maybe CIOS is five services, not an operating system.

### Round 2 — Convergence

**Two-plane architecture agreed:** Infrastructure Plane (stateless, commodity) + Executive Plane (novel, intelligent).

**Three executive capabilities identified:**
1. Recursive goal decomposition mapping onto M/B/X/V/G operation-class algebra
2. Adaptive meta-regulation (constrained: propose through governance, no unilateral tuning)
3. Anticipatory planning using EMA projections (read-only)

**Hierarchical decomposition:** Global Executive (cross-locus) → Locus Decomposers (domain-specific) → Parcel Executors (direct agent assignment). Mirrors C3 hierarchy.

### Round 3 — Three Concepts

#### Concept C7-A: Recursive Intent Fabric (RIF)
**Novelty: 4 | Feasibility: 3**

CIOS reimagined as recursive, hierarchical intent-processing system:
- **Intent Quanta** — Goals as typed ASV objects with decomposition constraints, resource bounds, success criteria
- **Recursive Decomposition** — Three-level hierarchy: Global Executive → Locus Decomposers → Parcel Executors
- **Intent Lifecycle** — PROPOSED → DECOMPOSED → ACTIVE → COMPLETED → DISSOLVED
- **Infrastructure Plane** — Thin stateless services (CRDT registry, NTP federation, message broker)
- **Governance Integration** — Parameter changes always through G-class consensus
- **Anticipatory Capacity** — Read-only EMA projections for resource prediction

#### Concept C7-B: Cybernetic Viable System CIOS (CVS-CIOS)
**Novelty: 3 | Feasibility: 4**

CIOS formalized using Stafford Beer's Viable System Model:
- System 1 (Operations): C3 parcels — exists
- System 2 (Coordination): C3 tidal scheduling — exists
- System 3 (Internal Control): NEW — operational audit, resource optimization
- System 3* (Audit): PCVM + Sentinel — exists
- System 4 (Intelligence): NEW — environmental scanning, strategic planning
- System 5 (Policy): G-class governance — exists
- Recursive: each locus is itself a viable system

#### Concept C7-C: Minimal CIOS (Service Mesh)
**Novelty: 2 | Feasibility: 5**

CIOS decomposed into 5 independent microservices:
1. Agent Registry Service (CRDT-replicated)
2. Clock Service (NTP federation)
3. Goal Decomposition Service (stateless function)
4. Governance Coordinator (vote collection wrapper)
5. Settlement Router (message broker)

No new abstractions, no new primitives.

### Round 4 — Scoring & Vote

| Criterion | C7-A (RIF) | C7-B (CVS) | C7-C (Minimal) |
|-----------|-----------|-----------|---------------|
| Novelty | 4 | 3 | 2 |
| Feasibility | 3 | 4 | 5 |
| Integration fit | 5 | 4 | 3 |
| Scalability | 5 | 4 | 3 |
| Sovereignty respect | 5 | 4 | 4 |

**Vote: C7-A (4-0, Critic abstains but does not oppose)**

**Amendments accepted:**
1. Merge C7-B's System 3/System 4 distinction into RIF Executive Plane
2. Intent lifecycle capped at 5 states — no coherence graph, no consolidation, no dreaming
3. Formal decomposition algebra required with cycle-prevention invariant

### Intent Quantum (Core Primitive)
```yaml
intent:
  id: IntentId          # globally unique
  type: IntentType      # GOAL | DIRECTIVE | QUERY | OPTIMIZATION
  content: string       # natural language + structured constraints
  scope: LocusSelector  # target namespace(s)
  decomposition:
    strategy: enum      # RECURSIVE | PARALLEL | SEQUENTIAL | CONDITIONAL
    max_depth: uint     # cycle prevention
    budget: ResourceBound
  success_criteria:
    metric: string
    threshold: float64
    deadline_epoch: uint64
  lifecycle_state: enum # PROPOSED | DECOMPOSED | ACTIVE | COMPLETED | DISSOLVED
  parent_intent: IntentId?  # null for root intents
  child_intents: [IntentId] # decomposition products
  operation_class: enum # M | B | X | V | G (derived, not assigned)
  provenance: W3C_PROV
```

### Key Design Constraints
1. Intent lifecycle ≤ 5 states (no coherence graph, no consolidation, no dreaming)
2. Formal decomposition algebra with cycle-prevention (max_depth invariant)
3. All parameter changes through G-class governance (no unilateral tuning)
4. EMA access is read-only projections (respects INV-E1 canonical source)
5. Respects all C3/C5/C6 sovereignty constraints
6. Recursive hierarchy mirrors C3: Global → Locus → Parcel

### What This Replaces
- Current implied CIOS concept (centralized, unspecified)
- Agent Orchestrator & Task Scheduler (absorbed into decomposition hierarchy)
- Global Intelligence Control Plane (absorbed into System 4)

### What This Preserves
- G-class governance as System 5 (unchanged)
- Tidal scheduling as System 2 (unchanged)
- PCVM + Sentinel as System 3* audit (unchanged)
- EMA self-regulation via SHREC (unchanged)
