# C8 — Settlement Plane Architecture Review and Reinvention
## Ideation Council Output

**Date:** 2026-03-10
**Stage:** IDEATION
**Status:** CONCEPT SELECTED — C8-A Deterministic Settlement Fabric (DSF)

---

## Domain Translator Brief

### Current State
Settlement Plane is ~60% correctly designed, ~40% needs redesign. Strong foundations (three-budget model, four-stream settlement, CSOs, deterministic computation) scattered across multiple documents without unified specification. Legacy blockchain assumptions don't fit the Tidal Noosphere substrate.

### What Works (Preserve)
1. Three-budget model (SB/PC/CS) — genuinely novel, prevents gaming
2. Four-stream settlement (compliance 40%, verification 40%, communication 10%, governance 10%)
3. CSO framework with formal conservation proofs
4. Deterministic settlement computation (INV-2)
5. Economic simulation scenarios E1-E7
6. Treasury-first issuance with hybrid governance

### What Doesn't Work (Redesign)
1. AIC as blockchain token — no chain exists in Tidal Noosphere
2. Task marketplace with external escrow — tasks come from RIF, not users
3. PoMS model fingerprint — impractical for LLM agents
4. Infrastructure provider compensation — unspecified
5. Settlement timing — single epoch frequency doesn't match multi-rate operations

---

## Council Deliberation

### Round 1-2: Consensus
Council unanimously agreed: improvement with substantial rearchitecting, not full reinvention. The three-budget model and four-stream settlement are proven economics. The substrate and staking models need replacement.

### Round 3: Three Concepts

#### C8-A: Deterministic Settlement Fabric (DSF) — N3/F4
- Preserve: three-budget, four-stream, CSOs, deterministic computation, E1-E7
- Redesign: AIC substrate (CRDT-replicated), staking (capability-weighted), task funding (intent-budgeted), infrastructure compensation (capacity market)
- New: Multi-rate settlement, capacity market, settlement determinism protocol

#### C8-B: Metabolic Economics Engine (MEE) — N4/F2
- Full reinvention: value as flow, no static balances, competitive dynamics
- Impractical: how do you slash a flow? How do you escrow a rate?

#### C8-C: Federated Settlement Protocol (FSP) — N3/F3
- Locus-local currencies with exchange protocol
- Risk: exchange rate instability, arbitrage attacks

### Round 4: Vote
**C8-A selected unanimously (5-0)**

### What's New in DSF (vs current Settlement Plane)

1. **AIC Substrate: CRDT-Replicated Ledger** — AIC balances stored as CRDT counters, not blockchain state. Deterministic settlement computation produces identical results on all nodes. No consensus needed for balance updates — they are computed outputs of the settlement function.

2. **Capability-Weighted Stake** — Replaces PoMS model fingerprint. Stake = AIC_collateral × capability_score, where capability_score = f(reputation, verification_track_record, claim_class_accuracy). No model fingerprint needed.

3. **Intent-Budgeted Settlement** — Replaces task marketplace escrow. RIF's intent resource_bounds serve as budget. Settlement computes rewards based on completion quality within budget. No external user escrow.

4. **Multi-Rate Settlement** — Different settlement frequencies for different operation classes:
   - Fast (per-epoch): B-class CSO resource accounting, PC decay
   - Standard (every N epochs): V-class verification rewards, compliance scoring
   - Slow (governance cycle): G-class parameter adjustments, treasury operations

5. **Capacity Market** — Formalized supply/demand mechanism for CSO resources. Infrastructure providers advertise available capacity. Agents bid with CS budget. Prices discovered through market clearing at epoch boundaries.

### What's Preserved

- Three-budget model (SB/PC/CS) — unchanged
- Four-stream settlement computation — unchanged
- CSO framework with conservation laws — unchanged
- Deterministic settlement invariant (INV-2) — strengthened
- Economic simulation scenarios E1-E7 — extended
- Treasury-first issuance — unchanged
- Graduated slashing — clarified (no model fingerprint dependency)
- Sentinel Graph integration — unchanged
