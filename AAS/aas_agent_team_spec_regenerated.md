# AAS Agent Team Specification (Regenerated)

This document defines the upgraded agent team structure used by AAS-RE.

## Core Teams

- Architecture Analysis Team
- Code Intelligence Team
- Governance Analysis Team
- Innovation Systems Team
- Meta-Architecture Team
- Simulation Team
- Architecture Pressure Testing Team
- Adversarial Architecture Review Team

## Architecture Pressure Testing Team

Responsibilities:

- scalable architecture simulation
- governance conflict simulation
- failure recovery simulation
- architecture evolution flexibility testing

Outputs:

- pressure-test reports
- stress traces
- failure recovery findings

## Adversarial Architecture Review Team

Responsibilities:

- hidden subsystem coupling detection
- governance deadlock probing
- architectural rigidity attacks
- complexity explosion analysis
- future evolution constraint forecasting

Outputs:

- adversarial invalidation reports
- residual risk findings
- veto evidence for councils and final selectors

## Coordination Model

All teams coordinate through the Coordination Kernel, which routes
outputs into councils, pressure testing, adversarial review, and task
generation.

## Objective

The upgraded team model supports bounded architecture research rather
than open-ended proposal generation.
