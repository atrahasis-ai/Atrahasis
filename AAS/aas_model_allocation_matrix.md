# AAS Model Allocation Matrix

## Atrahasis Agent System (AAS)

This document defines the model allocation policy for all agents
operating within the Atrahasis Agent System.

The goal of the allocation matrix is to ensure that each agent role uses
the most appropriate model tier for its task.

The system uses two primary models:

-   GPT-5.4
-   GPT-5.4 Thinking

------------------------------------------------------------------------

# Model Tier Definitions

## GPT-5.4

Use for:

-   system orchestration
-   architecture analysis
-   code intelligence
-   simulation
-   knowledge graph updates
-   task planning

Optimized for:

-   fast reasoning
-   large context workflows
-   agent orchestration

------------------------------------------------------------------------

## GPT-5.4 Thinking

Use for:

-   strategic reasoning
-   architecture invention
-   council deliberation
-   governance redesign
-   complex system analysis

Optimized for:

-   deep reasoning
-   long-horizon planning
-   multi-step architecture design

# Allocation Rules

1.  Thinking models must be used for strategic reasoning and
    architecture invention.
2.  Standard GPT-5.4 should be used for orchestration and analysis
    tasks.
3.  Councils must always run GPT-5.4 Thinking models.
4.  Kernel infrastructure agents must always run GPT-5.4.

------------------------------------------------------------------------

# Final Objective

The Model Allocation Matrix ensures efficient compute usage while
maintaining high reasoning quality across the Atrahasis Agent System.
