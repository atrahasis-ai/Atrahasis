# PART 1 — SPECIALIST ASSESSOR REPORTS

## 1.1 Technical Feasibility Assessor

**Score: 4.0/5**

C17 is technically feasible using established methods. Every component maps to known, proven technology:
- Feature extraction (BFE) uses standard statistical descriptors (histograms, moments, frequency distributions) — no novel computation required.
- Distance metrics are well-understood: Jensen-Shannon divergence, cosine distance, Wasserstein distance. All have efficient implementations in standard scientific computing libraries.
- LSH pre-filtering is a mature technique with decades of deployment at scale (Spotify recommendations, Google deduplication). The configuration (8 hash functions, 20 tables) is conservative and well-calibrated.
- Multi-task cross-correlation uses Pearson correlation and coefficient of variation — first-year statistics.
- Phase 2 contrastive learning uses a 3-layer Siamese MLP — trivial by modern ML standards.

**Challenges:**
- PCVM Behavioral VTD generation requires instrumenting the existing C5 pipeline to emit timing, structural, and resource telemetry alongside verification outputs. This is integration engineering, not research.
- The SEB task pool (100 tasks initially, 200 at Phase 2) requires curation effort. Task diversity and quality control are human-labor problems, not technical problems.
- Scaling validation: the LSH recall/precision targets (95%/10%) need empirical tuning at each phase boundary.

**Assessment:** No technical risks that would block deployment. The hardest problem is integration with PCVM (C5), which is well-scoped by DA-08.

## 1.2 Novelty Assessor

**Score: 3.5/5 — Novel Combination of Known Techniques**

**Novel elements:**
1. **Multi-modal behavioral fingerprinting of AI agents:** Prior art fingerprints LLMs via output distributions (Tay et al.) or watermarks (Kirchenbauer et al.). C17 fingerprints via 5 orthogonal behavioral modalities extracted from external observation (VTDs), without requiring model internals. No prior system combines temporal, structural, error, resource, and lexical modalities for AI agent attribution.
2. **Multi-task cross-correlation as anti-coincidence filter:** Using consistency of similarity across diverse task types to distinguish genuine same-origin from coincidental similarity is novel in the Sybil detection context. The drug panel testing analogy is apt.
3. **Adversary-weighted fusion:** Ordering modality weights by evasion difficulty (hardest-to-fake gets highest weight) is a defensive design principle not found in standard biometric fusion literature, where weights are typically optimized for accuracy alone.
4. **Behavioral trajectory comparison via DTW:** Comparing not just current behavioral profiles but how behavior evolves over time adds a temporal dimension to Sybil detection that is absent from existing approaches.

**Known elements:**
- Stylometry and authorship attribution (established field)
- Behavioral biometrics (keystroke dynamics, gait analysis)
- LSH for approximate nearest neighbor search
- Siamese networks for contrastive learning
- JSD, cosine distance, Wasserstein distance

**Prior art clearance:** No existing system performs multi-modal behavioral Sybil detection on AI agents using externally-observed VTDs. Individual techniques are well-established. The combination is novel.

## 1.3 Impact Assessor

**Score: 4.0/5 — High Impact Within Architecture**

C17 resolves OQ-2 from C14 (P0 priority, blocking Phase 1 entry). Without a functioning B(a_i, a_j), MCSD Layer 2 is a placeholder and Phase 1 cannot proceed. This makes C17 a critical-path dependency for the entire Atrahasis governance system.

**Direct impact:**
- Enables MCSD Layer 2 behavioral Sybil detection, completing the 4-layer defense stack.
- Enforces the 4.0x cost multiplier for Sybil attacks (C14 Section 10.2) — an adversary must maintain genuinely diverse architectures to evade behavioral comparison.
- Protects one-agent-one-vote governance integrity, which is the foundation of Atrahasis's democratic legitimacy.

**Systemic impact:**
- Behavioral VTDs generated by C17's PCVM integration become a data asset for C12 (AVAP behavioral analysis), C11 (CACT challenge probes), and future defensive systems.
- The SEB framework provides a standardized agent evaluation mechanism that could support governance qualification, capability assessment, and quality monitoring beyond Sybil detection.

**Limitation:** C17 addresses same-origin detection only. Same-controller-different-origin attacks (Attack 6: outsourced architecture) remain a Layer 3/4 responsibility. Impact is bounded by this scope.

## 1.4 Specification Completeness Assessor

**Score: 4.0/5 — Complete with Minor Gaps**

**Complete:**
- B(a_i, a_j) fully specified: 5 modalities, 27 features, 5 distance metrics, fusion formula, weight bounds, governance controls (DA-01)
- SEB design: 7 task categories, 30 tasks per instance, 100-200 task pool, randomization protocol, quarterly refresh (DA-02)
- Behavioral VTD schema: JSON structure with all 5 modality sections, privacy constraints, retention policy (DA-03)
- LSH configuration: hash family, table count, recall/precision targets, scaling estimates for Phases 1-3, incremental update protocol (DA-04)
- Graduated response: three zones (CLEAR/WATCH/FLAG), actions per zone, dispute resolution via Constitutional Tribunal, interpretable explanation format (DA-05)
- Temporal trajectory: rolling window, exponential decay, DTW comparison, 6th modality integration with weight redistribution (DA-06)
- Phase 2 contrastive model: Siamese MLP architecture, training data strategy, integration formula with alpha blending (DA-07)
- Cross-system integration: PCVM, AiSIA, CACT integration points specified (DA-08)

**Minor gaps:**
- No formal specification of the SEB task diversity metric — how is "diversity" of the 30-task random subset measured and enforced? What prevents a degenerate draw that over-represents one category?
- No specification for B(a_i, a_j) computation frequency — how often is the full pairwise comparison run? DA-04 describes incremental updates for new agents, but periodic recomputation for existing agents is not scheduled.
- The Phase 2 contrastive model's retraining schedule and validation protocol are described at the requirement level but lack a concrete cadence (quarterly? after N new training pairs?).
- No explicit treatment of the case where an agent legitimately changes architecture (e.g., migrates from GPT to Llama). How does this interact with behavioral trajectory comparison?

**Assessment:** The specification is sufficient to implement Phase 0-1. The gaps are operational details that can be resolved during integration without architectural changes.

## 1.5 Commercial Viability Assessor

**Score: 3.5/5 — Viable as Internal Infrastructure**

C17 is not a revenue-generating invention. It is defensive infrastructure that protects the economic value of Atrahasis's governance system. Its commercial viability is measured by cost-effectiveness, not revenue.

**Cost analysis:**
- Compute overhead: <2% of verification budget (from FEASIBILITY). At Phase 2 scale, the LSH + full B computation takes ~5 minutes per cycle. This is negligible compared to PCVM verification costs.
- SEB curation: requires dedicated effort (task creation, quality control, quarterly refresh). Estimated 0.5 FTE equivalent annually. Manageable.
- Storage: Behavioral VTDs are lightweight (~1KB per observation). At 100,000 agents x 200 observations = 20M VTDs = ~20GB. Trivial.

**Value protection:** MCSD's 4-layer defense protects one-agent-one-vote governance. A successful Sybil attack could compromise governance decisions worth orders of magnitude more than C17's operational cost. The cost-benefit ratio is strongly favorable.

**Adoption barrier:** C17 imposes no additional burden on legitimate agents beyond the SEB (30 tasks at onboarding, periodic renewal). This is a reasonable onboarding requirement that also serves as a capability demonstration.

# PART 3 — ASSESSMENT COUNCIL

## 3.1 Advocate

C17 delivers exactly what the architecture needs: a concrete, implementable algorithm for the previously-unspecified B(a_i, a_j) function that was blocking Phase 1 entry. The design is disciplined in scope — it solves the MCSD Layer 2 problem without scope creep into Layers 3 or 4.

Key merits:
- **Grounded in established science.** Every component (statistical distances, LSH, contrastive learning, DTW) has decades of theoretical foundation and production deployment. This is applied engineering, not speculative research.
- **Adversary-aware from the ground up.** The adversary-weighted fusion, randomized SEB, information asymmetry (agents cannot see their own fingerprints), and graduated response all reflect genuine adversarial thinking, validated by the 10-attack battery.
- **Phased deployment is realistic.** Phase 0-1 uses only statistical methods (no ML training data needed). Phase 2 adds contrastive learning only after sufficient labeled data accumulates. This avoids the cold-start trap.
- **Clean integration.** DA-08 maps every integration point with C5, C11, and C14. No new architectural dependencies or cross-layer conflicts.
- **Cost is negligible.** <2% of verification budget. The SEB doubles as a capability assessment tool, providing value beyond Sybil detection.

C17 should be approved. It is the most implementation-ready invention in the defense stack.

## 3.2 Skeptic

C17 is competent engineering, but several concerns deserve acknowledgment:

1. **False positive rate is entirely theoretical.** The graduated response (WATCH/FLAG) is a good design, but no empirical data exists on actual FPR in an AI agent population. If AI agents cluster into a small number of behavioral archetypes (because most are built on 3-4 foundation model families), the baseline similarity could be high enough to overwhelm the thresholds. The specification acknowledges this (F-02 in Pre-Mortem) but offers no concrete mitigation beyond "quarterly FPR audits."

2. **The 4.0x cost multiplier is inherited, not derived.** C17 claims to enforce C14's 4.0x cost multiplier for Sybil evasion, but this number comes from C14's economic analysis, not from C17's own security analysis. The actual evasion cost depends on how correlated behavioral features are across model families — if switching from GPT-4 to Llama-3 naturally produces B < 0.60, then the cost multiplier is 1.0x (just use a different model), not 4.0x.

3. **Lexical modality (w=0.10) may be dead weight.** The specification itself rates lexical features as "easiest to fake." At 10% weight, lexical contributes marginally to detection while adding feature extraction complexity and an attack surface (adversary can manipulate vocabulary to introduce noise). Removing it would simplify the system with minimal detection loss.

4. **Trajectory comparison (DA-06) is speculative.** The claim that same-origin agents drift in similar directions when updated is plausible but unvalidated. If agents are updated independently (different fine-tuning data, different RLHF), their trajectories could diverge even with shared origin.

These concerns are not blocking but should be monitored.

## 3.3 Arbiter

The Advocate's case is strong. C17 is a well-scoped, technically sound specification that fills a critical gap (OQ-2) in the Atrahasis architecture. The Skeptic raises legitimate concerns, none of which are architectural — they are calibration and validation issues that can be resolved empirically during Phase 0-1.

**Responses to Skeptic concerns:**
1. **FPR uncertainty:** Valid. This is inherent to any pre-deployment specification. The graduated response is specifically designed to handle FPR uncertainty gracefully — WATCH before FLAG prevents catastrophic false positives. Monitoring flag added.
2. **4.0x cost multiplier:** The multiplier is a system-level property of the 4-layer MCSD stack, not a Layer 2 guarantee alone. C17 contributes to the multiplier; it does not solely produce it. The specification is clear about Layer 2's scope limitations (Attack 6 acknowledged).
3. **Lexical dead weight:** The minimum weight bound (5%) ensures lexical cannot be fully removed without governance action. At 10%, the cost of maintaining it is trivial. If Phase 0-1 data confirms it adds no detection value, governance can reduce it to 5%.
4. **Trajectory speculation:** Correctly deferred to Phase 2. No Phase 0-1 decision depends on trajectory comparison.
